# macOS 应用打包与分发指南

既然开发已完成，您可以按照以下步骤将 **ScreenMarker Pro** 打包并分发给其他用户。

## 1. 使用 Xcode 进行归档 (Archiving)

这是官方推荐的打包方式，可以生成一个优化的 `.app` 文件夹。

1.  **选择目标设备**：在 Xcode 顶部的工具栏中，将目标设备选为 **"Any Mac (Apple Silicon, Intel)"**。
2.  **设置版本号**：在项目设置的 `General` 选项卡中，确保 `Version` (如 1.0) 和 `Build` (如 1) 已正确设置。
3.  **运行归档**：点击菜单栏：`Product > Archive`。
4.  **导出应用**：
    - 归档完成后，Xcode 会弹出 `Organizer` 窗口。
    - 点击右侧的 **"Distribute App"** 按钮。
    - 选择 **"Copy App"** (如果您没有开发者账号) 或 **"Direct Distribution"** (如果您有开发者账号并想进行公证)。
    - 按照提示选择保存位置，您将获得一个 `ScreenMarker Pro.app` 文件。

## 2. 创建磁盘映像 (DMG) - 选做

为了让用户安装更方便，您可以将 `.app` 放入一个 `.dmg` 文件中。

您可以直接在终端运行我为您准备的一个简单命令：

```bash
# 进入桌面创建一个临时文件夹
mkdir ~/Desktop/ScreenMarkerDist
cp -R "/path/to/your/ScreenMarker Pro.app" ~/Desktop/ScreenMarkerDist/

# 使用 hdiutil 创建 DMG
hdiutil create -volname "ScreenMarker Pro" -srcfolder ~/Desktop/ScreenMarkerDist -ov -format UDZO ~/Desktop/ScreenMarkerPro.dmg
```

## 3. 注意事项 (重要)

- **公证 (Notarization)**：如果您要将应用发给不认识的网友，且您有 Apple 开发者账号，建议在 Xcode 中进行公证。否则，其他人在打开应用时会遇到“无法验证开发者”的系统警告。
- **手动绕过警告**：如果没有公证，用户在第一次打开时需要：**右键点击应用 -> 选择“打开” -> 在弹窗中点击“仍要打开”**。
- **隐藏 Dock 图标**：我已经更新了 `Info.plist`，应用启动后将只显示在菜单栏，不会常驻 Dock 栏，这符合工具类软件的设计。

---

**打包建议**：如果您只是给同事或朋友使用，直接将导出的 `.app` 文件压缩成 `.zip` 发送即可。
