# 签名与公证指南

本文档提供macOS应用的代码签名和公证流程指南，确保应用可以在其他Mac上正常运行。

## 前置要求

1. **Apple开发者账号** - 需要付费的开发者账号（$99/年）
2. **开发者证书** - 在Xcode中配置
3. **App ID** - 在Apple Developer网站创建

## 第一步：配置代码签名

### 1.1 在Xcode中配置

1. 选择项目 → Targets → ScreenMarkerPro
2. 选择 **Signing & Capabilities** 选项卡
3. **Team**: 选择您的开发者团队
4. **Signing Certificate**: 选择 "Apple Development" 或 "Developer ID Application"
5. **Bundle Identifier**: `com.yourcompany.ScreenMarkerPro`（根据实际修改）

### 1.2 开启必要权限

在 **Signing & Capabilities** 中添加：

- ✅ **Hardened Runtime** - 已启用
- 可选：**Sandbox** - 根据需求（建议不开启，因为需要系统级权限）

## 第二步：构建发布版本

### 2.1 归档应用

```bash
# 方法1：使用Xcode
# Product → Archive

# 方法2：使用命令行
xcodebuild -project ScreenMarkerPro.xcodeproj \\
           -scheme ScreenMarkerPro \\
           -configuration Release \\
           -archivePath ./build/ScreenMarkerPro.xcarchive \\
           archive
```

### 2.2 导出应用

1. Window → Organizer
2. 选择刚才的Archive
3. 点击 **Distribute App**
4. 选择 **Developer ID** (用于分发到App Store外)
5. 选择导出选项：
   - ✅ Upload - 自动公证
   - 或 Export - 手动公证

## 第三步：公证（Notarization）

公证是Apple的安全验证流程，让您的应用可以绕过Gatekeeper的警告。

### 3.1 准备App-Specific Password

1. 访问 https://appleid.apple.com
2. 登录 → 安全 → App专用密码
3. 生成新密码并保存

### 3.2 创建公证配置

```bash
# 存储Apple ID密码到钥匙串
xcrun notarytool store-credentials "notarytool-profile" \\
  --apple-id "your-email@example.com" \\
  --team-id "YOUR_TEAM_ID" \\
  --password "xxxx-xxxx-xxxx-xxxx"
```

### 3.3 提交公证

```bash
# 1. 压缩应用
cd ./build
ditto -c -k --keepParent ScreenMarkerPro.app ScreenMarkerPro.zip

# 2. 提交公证
xcrun notarytool submit ScreenMarkerPro.zip \\
  --keychain-profile "notarytool-profile" \\
  --wait

# 3. 查看公证状态
xcrun notarytool log <submission-id> \\
  --keychain-profile "notarytool-profile"
```

### 3.4 装订公证票据

公证成功后，将票据附加到应用：

```bash
xcrun stapler staple ScreenMarkerPro.app
```

验证：

```bash
xcrun stapler validate ScreenMarkerPro.app
spctl -a -vvv -t install ScreenMarkerPro.app
```

## 第四步：分发

### 方法1：DMG安装包

```bash
# 使用create-dmg工具
# 安装：brew install create-dmg

create-dmg \\
  --volname "ScreenMarkerPro" \\
  --volicon "icon.icns" \\
  --window-pos 200 120 \\
  --window-size 800 400 \\
  --icon-size 100 \\
  --icon "ScreenMarkerPro.app" 200 190 \\
  --hide-extension "ScreenMarkerPro.app" \\
  --app-drop-link 600 185 \\
  "ScreenMarkerPro.dmg" \\
  "ScreenMarkerPro.app"
```

### 方法2：ZIP压缩包

```bash
ditto -c -k --keepParent ScreenMarkerPro.app ScreenMarkerPro.zip
```

## 第五步：验证安装

在**另一台Mac**上测试：

1. 下载/拷贝应用
2. 双击运行
3. 应该**不会**出现"无法验证开发者"警告
4. 正常显示权限请求对话框

## 常见问题

### Q1: 公证失败 - "The binary uses an SDK older than the 10.9 SDK"

**A**: 更新Xcode或设置`MACOSX_DEPLOYMENT_TARGET = 12.0`

### Q2: 运行时提示"已损坏"

**A**: 公证未成功或票据未装订。重新执行stapler步骤。

### Q3: 开机启动不工作

**A**:

- macOS 13+: 确保使用`SMAppService`
- macOS 12: 使用`SMLoginItemSetEnabled`
- 都需要签名应用

### Q4: 如何测试签名？

**A**:

```bash
codesign -dv --verbose=4 ScreenMarkerPro.app
codesign --verify --deep --strict ScreenMarkerPro.app
```

## 简化脚本（开发阶段）

创建`scripts/build-and-sign.sh`:

```bash
#!/bin/bash

# 构建
xcodebuild -project ScreenMarkerPro.xcodeproj \\
           -scheme ScreenMarkerPro \\
           -configuration Release \\
           clean build

# 签名
codesign --force --deep --sign "Developer ID Application: Your Name" \\
         build/Release/ScreenMarkerPro.app

# 验证
codesign --verify --verbose build/Release/ScreenMarkerPro.app

echo "✅ 构建和签名完成"
```
